Realistic Rendering

Deferred shading model and Physically based rendering

In order to ease the stress of multiple dynamic lights, Arcana Engine features a deferred lighting model. User-defined shaders output object properties to multiple render targets rather than just one color. Transparent objects can be rendered in a separate stage that maintains the blending capability,

Physically based rendering is designed to mimic light-surface interactions in the real world. Arcana Engine's PBR model lights objects based on surface microfacets, the principle of energy conservation, and a physically derived bidirectional reflectance distribution function.

Engine Features

For games and other graphics applications

Model and resource loading

Load models, materials, and other resources

Most resources can be defined with XML files. You can also create your own resources, specifying the parameters contained in the XML files.

Faster Than Light

Precomputed lightmaps, shadows, and indirect lighting

Lightmaps and global illumination textures are computed by the ArcanaFTL library, minimizing lag caused by having too many dynamic lights.

Massive Terrains

Procedurally generated heightmaps and textures

Render flat or spherical procedural terrains at an arbitrarily large scale. Terrain tile data is generated on the fly in real time.

Scene Structuring

World -> actors -> components

Easily fill a world with actors and actor components capable of almost any function. Actors are stored in a scene graph structure.
